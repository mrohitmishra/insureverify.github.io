You are a senior enterprise UX/UI designer.

Design a complete web and mobile dashboard UI for an Insurance Verification and Reporting Platform.
Focus only on product design, layout, user flow, and interaction.
Do NOT generate backend logic, APIs, or code.

### PRODUCT CONTEXT
The platform is used by a Master Vendor who works on behalf of multiple Insurance Companies.
Field Executives collect inspection data.
Back Office users validate data and generate reports.
Insurance Companies receive final reports.

### DESIGN GOALS
- Enterprise, professional, insurance-friendly look
- Clean, minimal, and scalable layout
- Role-based dashboards with clear navigation
- High data density but readable UI
- Designed for daily operational use

### ROLES TO DESIGN FOR

1. Admin Dashboard
- Master Vendor creation and management
- Platform health and audit visibility
- Minimal but authoritative UI

2. Master Vendor Dashboard
Design full navigation and screens for:
- User Management (Branch Manager, Team Lead, Back Office Executive, Field Executive)
- Client Management (Insurance Company, Client Branch, Zone, AO)
- Point Configuration & Report Builder
  - Excel upload based report structure
  - Field visibility (mandatory, optional, hidden)
  - Placeholder-based report sections
  - PDF and DOCX export options
- Branch and Zone Management
- Bank Account & Payment Setup
- Mail Configuration
- Profile, Change Password, Logout

3. Back Office Dashboard
Design screens for:
- Add New Case (manual entry from WhatsApp/email)
- Case List (branch-wise)
- Picked / In-Progress Cases (with lock indicator)
- MIS Dashboard (filters by date, status)
- Profile, Change Password, Logout

4. Field Executive Dashboard (Mobile-first)
Design simple and task-focused screens for:
- Todayâ€™s Cases
- Pending Cases
- Ongoing Cases
- Completed Cases
- Inspection form with photo upload and GPS indicator

5. Insurance Company Dashboard
Design screens for:
- Upload new case documents
- Select pincode / location
- View submitted cases
- Download final reports

### SPECIAL UI REQUIREMENTS
- Case status indicators (STARTED, PENDING, COMPLETED)
- Case lock indicator when a case is opened by someone
- Branch-based data visibility
- Report preview screen before final generation
- Clear confirmation and success states

### OUTPUT EXPECTATION
Provide:
- Complete dashboard screen list per role
- Navigation structure (sidebar / topbar)
- Page-wise layout description
- UX flow from login to logout
- Desktop design for web dashboards
- Mobile design for field executive app

Do not include any backend, database, or API design.
Focus only on UI/UX and product design.
